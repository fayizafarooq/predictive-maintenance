About This Project

Project overview, technologies used, methodology and academic context

Project Overview

This system applies machine learning to predict equipment failures before they occur, enabling proactive maintenance scheduling and reducing unplanned downtime. The project follows the CRISP-DM (Cross-Industry Standard Process for Data Mining) methodology across all phases from data understanding to deployment.

How It Works

  • 1,500-row synthetic dataset generated with realistic failure logic
  • 3 ML models trained: Logistic Regression, Random Forest, XGBoost
  • Daily script monitors 30 machines and updates predictions
  • Power BI dashboard visualises fleet health in real time

Technologies Used

Python Pandas Scikit-learn XGBoost Power BI Jupyter HTML/CSS/JS GitHub Pages

Academic Context

Final Year Project submitted in partial fulfilment of the requirements for the degree at Arab Open University — Bahrain Branch.


The project addresses a real-world industrial challenge, unplanned equipment failure, by applying supervised machine learning techniques to predict failures before they occur. Synthetic data was utilised due to the unavailability of proprietary industrial datasets.


The system demonstrates a complete data science pipeline, from raw data generation through to a live interactive dashboard, following the CRISP-DM methodology throughout.

Methodology — CRISP-DM

1
Business Understanding Define maintenance goals & failure criteria
2
Data Understanding Generate 1,500-row synthetic dataset
3
Data Preparation Scaling, train/test split (80/20)
4
Modelling Logistic Regression, Random Forest, XGBoost
5
Evaluation ROC curves, confusion matrices, GridSearchCV
6
Deployment Power BI dashboard & live prediction system
